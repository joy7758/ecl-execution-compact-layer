# ECL Feedback Intake Protocol v0.1

Status: initialized_no_feedback_claim

Public intake issue: https://github.com/joy7758/ecl-execution-compact-layer/issues/8

## What Counts

- A public GitHub issue from a third party.
- A public pull request from a third party.
- A public forum discussion where someone reports running `make demo`.
- A signed or attributable reviewer note that can be cited as feedback.

## What Does Not Count

- GitHub stars alone.
- Forks alone.
- Private conversation without permission to cite.
- Self-generated comments.
- AI-generated reviewer simulation.

## Required Fields

```text
feedback_id=
date=
source_url=
reporting_party=
what_was_run=
observed_result=
action_taken=
claim_boundary=
```

## Claim Boundary

Feedback is not adoption unless the external party uses ECL as a dependency or explicitly reports integration. Feedback is not validation unless the reported command and result are reproducible.

Opening the public intake issue is not feedback by itself. It only creates a public place where reproducibility reports can be recorded.
